Catching errors

try/catch

  • Ugly
  • Sync
  • Deoptimizes

guard()


function guard (fn) {
	return function () {
		try {
			return fn.apply(this, arguments);
		} catch (err) {
			logError(err);
			throw err;
		}
	};
}

Auto-wrapping is probably a bad idea.

Use promises

  • Wrap promise construction
  • Reject with new Error()
  • Don't forget to .catch(logError)
  • Stack traces an issue (unless native)
  • New: unhandledRejection
  • Problem: UI can't have a break in event loop

window.onerror
